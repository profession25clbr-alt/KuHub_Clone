Hay 3 cambios en 3 archivos para resolver un bug de tipos ENUM con Hibernate native queries.
El problema: Hibernate no puede asignar un parametro String a una columna PostgreSQL ENUM (estado_solicitud_type) en native queries.
La solucion: cambiar a JPQL con el tipo enum de Java que Hibernate si sabe manejar.

CAMBIO 1: SolicitudRepository.java (lineas ~236-242)
Quitar nativeQuery=true, cambiar la query a JPQL, cambiar el tipo del parametro de String a Solicitud.EstadoSolicitud.

ANTES:
    @Modifying
    @Query(value = """
            UPDATE solicitud
            SET estado_solicitud = :estado::estado_solicitud_type
            WHERE id_solicitud IN (:ids)
            """, nativeQuery = true)
    int updateMassiveStateSolicitation(@Param("ids") List<Integer> ids, @Param("estado") String estado);

DESPUES:
    @Modifying
    @Query("UPDATE Solicitud s SET s.estadoSolicitud = :estado WHERE s.idSolicitud IN (:ids)")
    int updateMassiveStateSolicitation(@Param("ids") List<Integer> ids, @Param("estado") Solicitud.EstadoSolicitud estado);
